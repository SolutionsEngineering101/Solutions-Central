# Skills Library

> Tracks the capabilities of the Solutions Engineering team.
> Use this to assign the right person to a request, identify gaps, and plan upskilling.

## Structure

```
skills/
├── README.md          — this file
├── matrix.md          — skills matrix: all members × all skill areas
└── domains/           — one file per skill domain with detail, resources, level guide
```

## Skill Levels

| Level | Label | Description |
|---|---|---|
| 0 | None | No exposure |
| 1 | Awareness | Knows the concept, hasn't applied it |
| 2 | Working | Can execute with guidance |
| 3 | Proficient | Independent, handles most cases |
| 4 | Expert | Go-to person, can train others |

## Updating Skills

- Edit `matrix.md` when someone levels up or a new skill is added
- Add a new file in `domains/` when a new skill area is introduced
- Commit: `skills: update [member name] — [skill]`
