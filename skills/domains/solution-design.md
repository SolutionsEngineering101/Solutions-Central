# Skill Domain: Solution Design

## What this covers
Writing PRDs, BRDs, Solution Complexity Documents (SCDs); building solution skeletons from client requests; scoping effort; identifying gaps and open questions.

## Level Guide

| Level | What you can do |
|---|---|
| 1 | Can fill in a template with guidance |
| 2 | Writes standard PRDs/BRDs independently for known use cases |
| 3 | Designs solutions for novel/complex requests; estimates complexity accurately |
| 4 | Sets standards for the team; reviews others' documents; interfaces directly with product/engineering |

## Key Resources
- Use `/prd` skill in Claude Code to generate a PRD
- Use `/brd` skill in Claude Code to generate a BRD
- `documents/complexity-mapping/` — complexity scoring guide (Low/Medium/High)
- `pre-built-solutions/blueprints/` — past solutions to reference
