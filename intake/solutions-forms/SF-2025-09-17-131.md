# Solution Request #131 - Sinyar
**Date:** 2025-09-17  |  **Status:** Open  |  **Complexity:** Low Complexity

## Subject
Solution request 138131::Acknowledgment of Milestone Notification Request – Sinyar

## Brief
Sinyar Milestone feature

## Problem Description
Sinyar has two entities in same platform. They want to send milestone notification using two entity name.

## Solution Given
Your concerns are completely valid, which is why I didn’t recommend changing the company name initially. But, for the time being, it would be practical to ask the client to suggest a name that can represent both entities without causing confusion.
This change would not impact the contract or any formal documentation - it would be limited to the Site Admin/backend configuration only.
In parallel, we will continue exploring the feasibility of implementing the 'Entity setup'.
Using the program name isn’t advisable either, as the company name - not the program name - is what gets passed from the backend for this specific placeholder. 
Also, the milestone feed post reads something like: “Congratulations on completing X years with [Company Name].”
If we were to use the program name here (e.g., for an R&R program that was launched only 6 months ago), it could be misleading - especially in cases where an employee is celebrating a much longer tenure, such as a 5-year work anniversary. The program name wouldn’t accurately reflect their time with the organization.
