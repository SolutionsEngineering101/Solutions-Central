# Solution Request #261 - Primus Partners
**Date:** 2026-05-21  |  **Status:** Solution Given Closed  |  **Complexity:** Not Set

## Subject
Solution Request : 261261: Primus Partners

## Brief
Configuration in award nomination workflow to validate nominee eligibility based on minimum tenure and previous award history

## Problem Description
The following validations are required:

Minimum Tenure Validation:

If an employee has not completed the minimum required tenure (e.g., 1 year) in the organization, the system should restrict the nomination process.

A pop-up message should appear informing the nominator that the employee is not eligible due to insufficient tenure.

This validation should apply across all award categories where tenure eligibility is mandatory.

Previous Award History Validation:

If an employee has already received a specific award within a restricted time period (e.g., in the last month or within the same quarter), the system should identify them as ineligible for nomination for awards that can only be granted once per quarter.

A pop-up notification should be displayed to the nominator highlighting the ineligibility based on award history.
