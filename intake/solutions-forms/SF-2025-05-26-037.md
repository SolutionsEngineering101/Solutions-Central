# Solution Request #37 - JSW Energy
**Date:** 2025-05-26  |  **Status:** Solution Given Closed  |  **Complexity:** Low Complexity

## Subject
Query 4337 JSW Energy Workflow

## Brief
The award workflow document is to be discussed.

## Problem Description
Employee:	View	eligible	awards,	submit	self/peer	nomination,	track	status
• Manager:	Submit	nominations,	endorse	with	input,	and view	submissions	from	their	team
• HOD:	Approve	or	comment	on	nominations,	flag	ineligible	cases
• Jury	Member:	View	assigned	nominations,	score	based	on	defined	criteria,	and add	optional	
comments
• Admin	(HR/IT):	Launch	award	cycles,	configure	award	types,	control	access	rights,	
track	completion,	generate	reports
• Team	Nominations:	Add	multiple	employees	under	one	award	submission,	and assign	roles.

## Solution Given
The workflow was discussed on call and the open items replied on mail.

Provision for ineligible cases: Apart from the advanced filters, we do not have eligibility mapping, and it is not in the roadmap either.
Provision for comments:
For the Standard Award workflow, if an Approver rejects an award, they have to give a reason for rejection.
For Panel Awards, if using the start rating method, up to two questions can be included for Approvers/Voters to answer that will be visible to the Winner Selector.
Jury Scoring Dashboard: As discussed, we do not have this, nor is it in the roadmap. However, we do have a Panel Nomination Report. The Client can use this data.
Gender-wise reports: We do not have any gender-wise filter for reports. However, if the Client shares gender information in the employee details, we can add this column to customize the Appreciation and Award Nomination Reports. Client HR can download the report and filter data accordingly.
Award specific form for nomination: While we do not have any such functionality, if the Client wishes to use a standard format, then they can use the filled form as an attachment with Business Justification while nominating from the platform.
