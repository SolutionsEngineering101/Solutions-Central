## Problem Description
1) Can there be a way where in multiple badges can be selected in one recognition
2) We have recently created one more category of hashtags but the category II hashtags are linked to category I hashtags. Is there a way these could operate independently

## Solution Given
Point 3-4: To product

Point 1-2: Within the non-monetary recognition workflow, there is an existing capability to add images. We can leverage this by introducing a predefined image library specifically configured for Argenx. This would function similarly to selecting stickers in a comments section.
Proposed approach:
The user selects a badge as usual.
Instead of adding secondary hashtags, the user selects additional attributes (e.g., quality, skill, or other criteria defined by the client) in the form of images from the predefined library.

These selected images would appear alongside the badge in the feed carousel.

This approach avoids any global frontend changes while still meeting the client’s requirement to visually represent multiple recognition criteria.
