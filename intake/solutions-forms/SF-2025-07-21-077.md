# Solution Request #77 - MPAC
**Date:** 2025-07-21  |  **Status:** Solution Given Closed  |  **Complexity:** Medium Complexity

## Subject
Solution request 8377::Anonymous User Profile Picture Display

## Brief
Anonymous User Profile Picture Display

## Problem Description
MPAC has flagged an issue regarding how anonymous users are displayed when recognitions are shared on the platform.

Currently, when a recognition includes multiple recipients and one of them has enabled privacy settings (e.g., "Require my consent before recognitions are shown"), their name appears as "Anonymous User"—which is expected. However, if that anonymous user has a profile picture, the system currently shows it in a blurred format.

MPAC would prefer a change to this behavior:

Instead of displaying a blurred profile image for anonymous users, the platform should display the default avatar (as if no profile picture was uploaded at all).

This change would:

Avoid confusion caused by the blurred image (which draws unnecessary attention),

Align better with user privacy expectations,

Provide a cleaner and more consistent user experience.

Let us know if this change is possible and if any additional information is needed.

## Solution Given
There are three scenarios for profile picture display:

Profile Picture Uploaded (No Consent Restrictions):
The image is visible to all users.

Consent Enabled (User Opts for Anonymity):
The user appears as an "Anonymous User" with a randomly selected blurred placeholder image. These images are not linked to actual user photos and are used to maintain privacy, similar to practices on platforms like LinkedIn.

No Profile Picture Uploaded:
A default placeholder image is shown.

This approach ensures privacy while maintaining a clear user interface distinction.
