# Solution Request #8 - PhonePe
**Date:** 2025-04-03  |  **Status:** Solution Given Closed  |  **Complexity:** Medium Complexity

## Subject
PhonePe Milestone Solution

## Brief
The client wishes to have the milestone catalogue with an option to target a specific set of employees, we need a work around on the request as the client wants this to be available for the employees by May'25

## Problem Description
The client wants to create a system in the milestone catalogue where they can target a specific set of employees, they are planning to give out Vouchers (Hotel & Eletronics) for all the employees along with that they want to add a Cash Out option for the BDA employees, the idea behind this is since the salary band is low and providing them with the cash out option so that they can use the value as per their preference.

## Solution Given
Initial response(ppt) was given through Team. The second response through mail.

To address this need, we propose leveraging the Multiple-Wallets Feature(deck attached in link), which allows PhonePe to create two separate wallets:​

Wallet A: With CTC-based cash-out option.​
Wallet B: No cash-out option for higher CTC employees.​

Each employee will be awarded Service Anniversary points in their respective wallet as per their eligibility through CSV upload.​

They did not go ahead with this approach. A second solution was suggested to implement two separate companies one with cash-out option and one without. Doing this would limit the possibility of SSO in future also. Shared R&R workflows would also not be possible.

Client onboarded
