# Solution Request #40 - Weaver (Team USA)
**Date:** 2025-05-30  |  **Status:** Solution Given Closed  |  **Complexity:** Low Complexity

## Subject
Solution Request 4640 Clarification on Award Eligibility Restrictions and Admin Visibility for Recipient Limits

## Brief
Client seeks clarity on role-based award restrictions and admin visibility into recipient award limits before assignment.

## Problem Description
Client Query:
Can we restrict our partners or some users from receiving Monetary Award?

Our Response:
We can restrict Awards by Department, Gender, Grade Group, and Entity through the Advanced Filtering option on the dashboard. If a user (or a group of users) does not qualify as per these restrictions, then they are automatically restricted completely. When such users are selected in the award workflow, no awards will be displayed since the criteria are not met.

Client’s follow-up inputs and specific requirements (along with my internal notes for clarity):

1. Interns/Contractors – Should be completely excluded from the system. (possible)

2. Partners – Should not be eligible to receive awards but can distribute points/awards. (Need clarity)

3. Eligibility to Distribute Points – Only users at Senior level and above should have the ability to distribute points. Those below Senior level should only be able to receive them. (possible)

Client Query:
While giving an award, is there any way to check how many awards the recipient is still eligible to receive as per the set limits?

Our Response:
No, it cannot be checked through the system. Once the recipient crosses the set limits and someone tries to award them, the system will throw an error message.

Client’s follow-up question:
"I know the employee will not have this visibility, but will anyone who has admin rights be able to see when a recipient has met their award limit for that quarter?" (Need clarity)

## Solution Given
Client’s follow-up inputs and specific requirements:

Interns/Contractors - Should be completely excluded from the system.
Input: Yes, this is feasible and can be implemented.

Partners - Should not be eligible to receive awards but can distribute points/awards. 
Input: Additional clarification is required on the role and access level of Partners.

Eligibility to Distribute Points - Only users at Senior level and above should have the ability to distribute points. Those below Senior level should only be able to receive them. 
Input: Yes, this is accurate. We can restrict nominators at Grade Level to distribute points as awards.

"I know the employee will not have this visibility, but will anyone who has admin rights be able to see when a recipient has met their award limit for that quarter?" 
Input: Currently, there is no functionality available for admins to view the award quota status on the recipient’s end.
