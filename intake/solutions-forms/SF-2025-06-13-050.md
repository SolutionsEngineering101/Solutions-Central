# Solution Request #50 - MPAC
**Date:** 2025-06-13  |  **Status:** No Response Closed  |  **Complexity:** Low Complexity

## Subject
Solution Request 5650 MPAC Consent

## Brief
When a group of employees is recognized and if one user ‘hides’ the post, it removes the post for all of the employees who were recognized in the group. 

Others should be able to view it

## Problem Description
We received a feature request from the client where they would like users to have the ability to hide their own recognition posts from the main feed.

For example, if Garima, Kamalika, and Shibani are appreciated in the same post, Garima should be able to hide the post from the feed only for herself, while the post should continue to remain visible for Kamalika and Shibani.

Kindly let us know the feasibility of this request and any potential limitations or implications.

## Solution Given
Our understanding: 
If one recipient of a team award hides the post, it becomes hidden for all other recipients as well.
Clarification:
The system is designed and tested to hide the post only for the specific recipient who chooses to hide it. If other recipients have consented to publish, the post should still appear in their feed as expected.
If you've observed behavior that differs from this, could you please share specific examples so we can investigate further?
