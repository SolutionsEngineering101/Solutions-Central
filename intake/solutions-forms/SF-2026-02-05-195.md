Solution request::199195::Award Specific Budget Wallet for Budget Holders (Manager's Budget Wallet)

## Brief
Award Specific Budget Wallet for Budget Holders (Manager's Budget Wallet)

## Problem Description
Can we evaluate the requirement for Award-Specific Budgets?

For example, if a company has five awards and, for each award, the budget holder maintains five separate wallets. We currently support this functionality in a backend-driven manner, where wallets are aligned to specific awards.

However, could you please assess the feasibility of extending this capability to the budget holder level with visibility in the frontend as well? The reason for this request is that we have received multiple client requirements where a fixed budget needs to be allocated per award. At present, we manage this using quota + No Budget, but this approach does not always yield the desired outcome.

## Solution Given
The request to introduce award‑specific budget wallets for budget holders, is a capability currently supported only through backend-controlled configurations. Implementing this at the frontend and budget‑holder level would require significant architectural, UI, and reporting changes, making it a potential new product feature rather than a simple enhancement.So ticket is forwarded to Product team. Awaiting Product’s evaluation and confirmation on next steps from Shasanka/Nayan.
