# Solution Request #182 - Infosys
**Date:** 2025-12-17  |  **Status:** Solution Given Closed  |  **Complexity:** Not Set

## Brief
Points transfer module-backend calculation

## Problem Description
We need to enable the Points Transfer module for Infosys. However, the client would like to understand the in-depth details of how the calculation/conversion is handled by the system.

I have informed the client that the system calculates the value based on the current exchange rate. As per my discussion with Subhendu, we update our exchange rates on a daily basis.

The client has raised concerns with this approach, as fluctuations in the exchange rate may impact their budget. They have therefore requested us to check whether it is possible to incorporate static exchange values or explore any alternative approach, as they are not willing to proceed with the current process.

## Solution Given
Exchange Rate Handling
The system uses real-time exchange rates, updated daily, to convert points between currencies during transfers.
Static exchange rates are not supported, as this could lead to inconsistencies and compliance issues.
Impact on Billing
Please note that exchange rate fluctuations do not affect client billing.
Billing is always based on the original currency of allocation, regardless of where the points are transferred.
When points are transferred, the system treats them as:
Redeemed in the source country → billed to Infosys at the original currency value.
Credited in the destination country → marked as non-billable points for future redemption.
Example Scenario
Suppose 1,000 points allocated in India (worth ₹1,000) are transferred to a user in the USA.
At the time of transfer:
The system considers the 1,000 points redeemed in India and bills Infosys ₹1,000.
The user in the USA now sees equivalent points in their account.
When they redeem these points in the USA, Infosys is not billed again, as these points are flagged as non-billable.
Key Assurance
Real-time exchange rates are necessary for accurate conversion.
Client budget remains unaffected because billing is tied to the original allocation currency.
Points transfer is purely a user experience feature, not an additional financial liability.
