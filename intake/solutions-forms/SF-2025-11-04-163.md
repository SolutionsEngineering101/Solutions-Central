# Solution Request #163 - Muthoot Capital Services Limited
**Date:** 2025-11-04  |  **Status:** Solution Given Closed  |  **Complexity:** Medium Complexity

## Subject
Solution Request 166163::Feasibility Discussion,Quota Return Logic for Declined Nominations (Muthoot Capital)

## Brief
When a nomination submitted by a nominator is declined, the deducted quota gets back to the wallet of the nominator. The client, MCSL has asked not to return it to the nominator. We need to find out if it is technically feasible.

## Problem Description
When Quota concept is applied to the nominator, the deducts the allocated quota form the nominators wallet when the nomination is submitted but the same number of quota gets back to the wallet if the nomination is declined. The MCSL team has prefidefined periods for each nomination and award. So, for example if four nominations are submitted where one gets approved and others are declined, the quota for three declined awards should should not get back to the nominator.

## Solution Given
When nominations are sent to the approver, the approver will approve only the eligible nominations and will not decline the non-eligible ones.
By doing so, the quota corresponding to the declined nominations will not be reallocated to the nominator.
This approach aligns with the client’s requirement that once a nominator submits a nomination for an award, the quota for that award should not be returned or reallocated to the nominator.
Optional Implementation
If the approver wishes to clear the list of pending nominations, they can do so at the end of the financial year. For instance, on 31st March each year, the approver may decline all pending nominations for that financial year.
On 1st April each year, the system will automatically deallocate any remaining quota from the nominators’ wallets whether it’s unused quota or quota that was reallocated after nomination declines.
This will ensure that the quota balance is reset to zero for all nominators at the start of the new financial year.
Since the client uploads quota manually via CSV, once the quota has been nullified, the client admin can proceed to allocate fresh quota for the new financial year.
It is recommended that the client provides a specific date on which our scheduler should run to deallocate quota at the end of the financial year — for example, midnight on 1st April.
