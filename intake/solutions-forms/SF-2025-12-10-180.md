## Subject
Solution request::183180::Clarification Required on Client Request for User Acceptance Agreement

## Brief
User Acceptance Agreement for Budget Holders

## Problem Description
The client has requested that since this is the 1st time that designated stakeholders will be provided a budget for recognition, can we create some sort of user acceptance agreement which will go only to budget holders. The user acceptance will be on the lines of - "I understand that the budget must be handled fairly and responsibly etc." - and these users need to click "Accept and Continue" to move forward. Will it be possible to add such a user acceptance agreement:

For the 1st time that a budget holder logs into the platform
For every time that the budget holder receives a fresh budget (ie, every quarter)

## Solution Given
Clarification nedded before proceeding Should the message be Informational only, or
A pop-up requiring user action?
If a pop-up:
Should it include only “Accept and Continue”, or also “Cancel” / “Reject”?
If user does not accept:
Should they be logged out or blocked from access?
Budget holder identification:
Confirm who will be the budget holder (nominator or approver).
Budget allocation process:
Confirm how allocation works for Servier.
