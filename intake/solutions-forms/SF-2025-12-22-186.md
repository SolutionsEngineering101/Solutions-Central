# Solution Request #186 - All Clients

## Subject
Solution request 189186::Proposed Approaches to Improve CSV Data Upload Accuracy

## Brief
Proposed Approaches to Improve CSV Data Upload Accuracy

## Problem Description
We have observed multiple instances of incorrect data uploads, which have resulted in duplicate or incorrect entries in the system. This, in turn, is causing mismatches and requires our team to spend considerable manual effort to identify and resolve the discrepancies.
We have encountered several such cases recently, particularly with Wipro and Celebal, where repeated upload errors have affected overall data accuracy and processing time.
To mitigate these issues and ensure smoother operations, we have outlined a set of approaches and best practices to improve the CSV upload process and reduce data anomalies. These steps aim to prevent future errors, enhance efficiency, and minimise manual intervention.
Please review the proposed approaches and share any additional inputs or suggestions that can further strengthen the process.

## Solution Given
We have completed the analysis and here is a summary of our findings:

Dry-Run / Preview Screen:
A preview option already exists in the Bulk Points Upload section. Adding a similar preview feature to the Employee Upload section is feasible and will be taken up in an upcoming sprint.

Smart Upload Validator:
The system already supports comprehensive validations. Immediate errors are shown for incorrect formats or missing primary data. Partial file processing is enabled—error rows are skipped, and details are emailed to designated recipients. For common formatting issues (like spaces or uppercase email IDs), the platform automatically standardizes the data before upload.

Two-Step Approval:
Two-step approval currently exists for Bulk Points Upload. Extending this to Employee Uploads would require architectural changes and may affect existing clients by requiring new uploader–approver configurations, potentially disrupting current workflows.
