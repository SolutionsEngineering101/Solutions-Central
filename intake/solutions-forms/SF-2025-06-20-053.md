# Solution Request #53 - Omron Automation Private Limited
**Date:** 2025-06-20  |  **Status:** Solution Given Closed  |  **Complexity:** Medium Complexity

## Subject
Omron Automation | Feasibility Check for different scenarios of Panel Awards

## Brief
Feasibility of Queries/Scenarios raised for Panel Awards related to:

1. Nomination based on levels/grades in the organization
2. Nomination capacity of Nominator 
3. Visibility of sections like My Panel Vote, Announce Winner & Panel Nomination Review in the To-Do List section

## Problem Description
This is in regards to one of my clients Omron Automation. They have four Panel Awards: 

Omron Innovista
Omron Collabtribe
Omron Dynamo
Omron CX Takumi

Workflow: 

Nominee = All Employee except Reporting Manager+1 (Cross functional nominations applied)

Nominator (All People Managers/Reporting managers)>>Reviewer (Reporting Manager of Nominator)>>Panel Voters>>Winner Selector

Queries: 

1. Nominators can nominate employees of the same level or below, but cannot nominate their Reporting Manager (say LI) and L2 level employees. Client won't share the ranks/grade groups of the organization, however, can share the user specific details of the above in Employee Details sheet. Is it feasible?

2. A Nominator cannot nominate one employee twice for one award, but can nominate one employee for all the 4 awards or for multiple awards. Is this feasible and how can we incorporate the same?

3. While submitting the UAT we see that the users are able to see multiple options relevant to the Panel awards such 'My Panel Vote, 'Announce Winner'. For example, if someone is the 'Winner Selector' he/she is able to see the 'My Panel Vote' option. Same way, if some is the Panel Voter, he/she is able to see 'Announce Winner' option along with other options of Panel awards. Kindly confirm if it is an expected behaviour.

## Solution Given
You can configure the Panel Award as a Team Panel to support both individual and team nominations. An advanced filter can be applied to limit the number of members in a team nomination (minimum of 1 and a maximum based on client requirements).

An employee may receive multiple nominations for the same Panel Award from different nominators. Each nomination is treated as a separate entry by the system.

The reviewer has the authority to evaluate and decide whether multiple nominations for the same employee should be forwarded to the panel or rejected.

If multiple nominations are forwarded, the Winner Selector has the discretion to determine whether the same employee can be awarded more than once. Please note that this logic cannot be enforced through system validation.

Regarding the issue with multiple roles, please try creating new users and awards to validate the hypothesis. If the issue persists, the Tech team will need to perform a root cause analysis (RCA).
