**Date:** 2025-07-16  |  **Status:** Solution Given Closed  |  **Complexity:** Medium Complexity

## Subject
Solution request 7973 OTP authentication for redemption

## Brief
TCS NA has signed with us, and I want to get your thoughts on executing a potential solution.
 
Situation: TCS is implementing a gift card redemption only program with us. Since this program is only for active employees, they want to make sure an employee is authenticated before making a redemption.
 
The authentication will work by sending an OTP to the employee, which they will enter on VC platform before proceeding to redeem their points for the Gift Card.

Execution: Since this has not been implemented for any of our clients before. How would we execute this OTP process?

## Problem Description
Situation: TCS is implementing a gift card redemption only program with us. Since this program is only for active employees, they want to make sure an employee is authenticated before making a redemption.
 
The authentication will work by sending an OTP to the employee, which they will enter on VC platform before proceeding to redeem their points for the Gift Card.
 
Their goal is to filter out terminated employees with the OTP process

Employee Journey
1. Employee earns Gems via the Ultimatix >TCS Gems Rewards platform.
2. Employee logs into the Vantage Circle Portal via a secure link or platform.
3. Employee creates or verifies an account.
4. Employee redeems TCS Gems (converted into Vantage Points) on available offerings.
5. Employees receive digital vouchers or instructions for experiences within 24 hours.

## Solution Given
Our understanding of the problem statement: The client requires OTP-based authentication to ensure that terminated employees cannot make redemptions or access the platform.

Proposed solution: Users are authenticated during login. If a user’s termination status is communicated to us through the HRMS system, that user will be prevented from logging in and making redemptions on the platform.

The SOW shared does not clearly state any HRMS integration. It mentions "Facilitate data sharing (as per TCS data privacy policies) to enable account creation and verification." 

Could you please confirm if there are any additional aspects to this requirement?

User Account Creation & Authentication

Adding users as and when the file is received from the client is possible. When the HR Admin(or SPOC) performs a CSV points upload:

New users will receive an email prompting them to register and redeem their points.
Existing users will be notified about the point upload.

This fulfills the user registration criterion sought by the client.

Authentication & Session Management

We can enable OTP-only login for the Web app with minimal development effort, rather than implementing a full-fledged two-factor authentication (2FA) mechanism.

However, mobile app will need to be disabled for this client, as the app does not currently support hiding the password login option.
