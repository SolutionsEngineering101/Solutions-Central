# Solution Request #184 - LTTS, Tata Motors & BOsch BGSW
**Date:** 2025-12-18  |  **Status:** To Product Closed  |  **Complexity:** High

## Subject
Country-Level Report Access for HR Admins – Feasibility & Next Steps

## Brief
Require Location & Department-specific reports for particular HR's who can download the report directly from the dashboard without accessing other options in the dashboard. They should be able to access only reports section.

## Problem Description
Require Location & Department-specific reports for particular HR's who can download the report directly from the dashboard without accessing other options in the dashboard. They should be able to access only reports section.

## Solution Given
Hi Ankit,
Regarding country-level access for HR Admins, where an admin can view reports only for the countries assigned to them, this has been discussed with @Armaan. This requirement will be addressed as part of the ongoing initiative from Product team, to provide restricted access to only reports for an admin.
Currently, country assignment for HR Admins is not an existing feature and will need to be developed. This enhancement must also remain compatible with the existing Reports filter UI. At present, report filters operate independently and do not support hierarchical relationships.
This means that if an admin is assigned:
Countries: A, B, C
Departments: D1, D2, D3
Then:
The Country filter will show A, B, C
The Department filter will show D1, D2, D3
However, a hierarchical mapping between country and department for an admin—such as:
Country A → Departments D1, D2
Country B → Departments D2, D3
Country C → Department D3
is not supported with the current Reports filter UI.
As a next step, @Armaan will validate the feasibility of this approach with the technical team, define the appropriate technical approach for country mapping, and plan the work into an upcoming sprint.
@Armaan, please help move this forward and share the expected timelines after discussing with the tech team.
Please note: This request will be addressed by the Product team as a part of their ongoing user level reports access project.
