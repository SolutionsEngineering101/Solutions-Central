# Solution Request #199 - Kotak Mahindra bank
**Date:** 2026-02-09  |  **Status:** Solution Given Closed  |  **Complexity:** Medium Complexity

## Subject
Solution request203199::Kotak - Budget/Quota configuration

## Brief
Kotak - Budget/Qouta configuration

## Problem Description
Kotak has an already existing Panel Award configured with 7 selectors and active budgets.
A new team (already on the platform) will be using monetary quota awards for the first time.

There are three award categories: 
•Individual monthly award, which is auto-approved for 1,000 points each, and a maximum of two awards can be given monthly. So, 2000 points per nominator should be allocated per month. This should lapse at the end of each month.
 
•1 quarterly individual award of 3000 points each, and a maximum of 3 awards can be given per quarter, so 9000 points per approver should be allocated per quarter.
 
•2 quarterly team awards of 10,000 points can be approved by each approver per quarter, so 20,000 points per approver should be allocated per quarter. Awards can be given to min 3 max 15 users, and the amount needs to be divided amongst the winners
 
Award Requirements
1.	Monthly Individual Award
	-Value: ₹1,000 (fixed)
	-Approval: Auto-approved
	-Eligible users: 57 (selected via email)
	-Frequency: Each user can give 2 awards per month
2.	Quarterly Individual Award
	-Value: ₹3,000 (fixed)
	-Approval: One-level approval
	-Flow: Nominator selects the approver
	-Limit: Each approver can approve 3 awards per quarter
	-Eligible users: 7 users (same from above)
3.	Quarterly Team Award
	-Value: ₹10,000 (fixed)
	-Approval & users: Same as Quarterly Individual Award
	-Limit: Each approver can approve 2 awards per quarter
	-Eligible users: 7 users (same from above)
	-Award can be given to min 3 max 15 users, and the amount needs to be divided amongst the winners


Constraint / Issue
•	Enabling auto budget allocation with fresh allocation for these quota awards causes:
	-Existing Panel Award budgets to lapse, which is not acceptable.

•	Displaying quarterly budget balances to monthly award givers is not acceptable, as it creates confusion (users may assume they can give more than 2 monthly awards).
 
Key Requirements
•	Existing Panel Award budgets must remain untouched
•	Monthly award givers should not see any monetary budget
•	Users should be restricted strictly by:
	-Award category
	-Frequency (monthly/quarterly)
	-Count limits (not budget balance)
 
Ask / Support Needed
•	Configuration approach that allows:
	-Quota-based award limits by category & frequency
	-No visible budget to monthly users
	-No impact on existing Panel Award budgets

Please advise on the best feasible configuration or workaround within the platform, for which monthly budgets get relapsed every month, and quarterly budget relapses every 3 months for concerned users. 

Thanks.
