# Solution Request #76 - Edelman financial engines
**Date:** 2025-07-17  |  **Status:** Solution Given Closed  |  **Complexity:** Low Complexity

## Subject
Solution request 8276 work anniversary location Edelman

## Brief
Request to Remove Location from Work Anniversary Posts

## Problem Description
Edelman has raised a request regarding the "Share Your Thoughts" section displayed for work anniversaries.

Currently, on the left-hand side below the anniversary badge, the text reads something like:
"Rakesh Das has been a part of Vantage Circle for 1 year as of July 22, 2025, and is currently working in the CX (R&R) Department, based in India."
On the right-hand side, users can write their messages.

They would like to know if it's possible to remove the location reference (e.g., “based in India,” “based in USA,” etc.) from this section.

Please let me know if this can be customized

## Solution Given
We have received your query. Before we begin evaluating the possibility of removing the location reference, could you please share the client's rationale behind this request?
Note: As a standard practice, please ensure that any client requests are accompanied by a clear justification. This helps us better understand the context and provide more effective solutions.
One helpful approach is to ask the client what specific challenges or constraints they are facing that led to this request.

We discussed this with the Tech team. The current code is designed such that if a user’s city is not available in the database, it will not be displayed on the Share Your Thoughts page (specifically for the work anniversary location).
I have a follow-up question:
Do any workflows, feed filters, employee search, or other platform features currently utilize the city parameter for Edelman?
If the city parameter isn’t being used elsewhere, we can consider removing this information from the client records altogether. This would allow the city data to naturally stop appearing on the platform without requiring any code changes and without impacting any existing workflows.
As for the country parameter, it is a mandatory field, as all awards and their corresponding values are configured and mapped at the country level.
