# Solution Request #150 - Weaver and Tidwell India LLP
**Date:** 2025-10-14  |  **Status:** Solution Given Closed  |  **Complexity:** Low complexity

## Subject
Solution request 155150::Weaver- Notification Message for Previously Awarded Nominees

## Brief
Weaver- Notification Message for Previously Awarded Nominees

## Problem Description
This is a requirement for the company Weaver. Weaver has cross-department awards. They want nominators to see a message indicating if a nominee has already received any awards in the past ( for a quarter ).

Whenever a nominator initiates a new nomination, a message should appear below stating:
“This nominee has already received an award.” (if applicable)

However, this message should serve only as an informational warning — it should not restrict or prevent the nominator from proceeding with the nomination if they still wish to do so. In essence, the message is intended as a prompt for awareness, not a limitation.

## Solution Given
The current system architecture does not support passive notifications. Key points include:
Validation Flow Limitation: The system treats validations as hard stops; if a nominee has already received an award, the nomination process must be restarted.
Flexibility in Nominations: Hard restrictions could block valid repeat recognitions, which goes against the inclusive intent of the program.
Performance Concerns: Real-time checks for prior awards would increase system complexity and load.
Alternative Option: If the goal is to restrict awards within a specific timeframe (e.g., one per quarter), that can be enforced via backend rules.

Passive alerts without blocking nominations are not feasible in the current setup.
