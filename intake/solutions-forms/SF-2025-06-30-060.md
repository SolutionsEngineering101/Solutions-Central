# Solution Request #60 - MPAC
**Date:** 2025-06-30  |  **Status:** Solution Given Closed  |  **Complexity:** High Complexity

## Subject
Solution Request 6660 Private Recognition MPAC

## Brief
User Feedback: Clarifying Private Recognition for Receivers

## Problem Description
We’ve noticed that when a user receives a private recognition, there isn’t a clear indication on their end that the recognition is private. This can sometimes lead to confusion.

Would it be possible to have something in place—such as a label or visual indicator—to let the recipient know that the recognition is private?

## Solution Given
It was agreed to add a diclamer to the email suggesting that the recognition received is private. Additionally the same has been implemented for all Award and Badge history section on the platform too.
