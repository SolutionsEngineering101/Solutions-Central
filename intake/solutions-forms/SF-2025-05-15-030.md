# Solution Request #30 - Aspire Systems
**Date:** 2025-05-15  |  **Status:** Open  |  **Complexity:** Medium Complexity

## Subject
SOLI configuration challenges and workaround | Aspire Systems

## Brief
We are currently working on a client implementation that involves configuring awards across multiple geographic locations, with USD, INR, and other countries currency currency values.

During the setup, we encountered a challenge where certain awards have the same USD value but different INR equivalents. For example:

Star Gladiator Award: USD 500 / INR 5,000

10 Year Anniversary Award: USD 500 / INR 10,000

As per SOLI configuration rules, a unique USD value can not be mapped to different INR values.

We’ve explained this limitation to the client; however, they are firm in their stance. They’ve indicated that modifying the INR or USD values is not an option, as it would impact long-standing internal frameworks and require high-level approvals that are not feasible at this stage. They’ve requested that we proceed with configuring the awards exactly as specified.

Given this constraint, we seek your guidance on how best to proceed. Are there any workarounds or possible exceptions within the SOLI framework that could accommodate this requirement? Alternatively, would a customization or override be possible to allow duplicate USD values with different INR equivalents?

Appreciate your input on how we can move forward while aligning with both the system’s constraints and the client’s requirements.

## Solution Given
The mapping of denomination to value has to be unique and it can not be altered. The complete list of Award values has been asked from the Client to find a workaround by mapping closest or next higher/lower denomination to the required value if possible.

Additionally, country-wise tenure award configuration development (like catalog) has been initiated. It may be be availble for the go live of this client.
