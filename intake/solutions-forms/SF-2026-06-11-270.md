# Solution Request #270 - Tata Technologies
**Date:** 2026-06-11  |  **Status:** Unknown  |  **Complexity:** Not Set

## Brief
Possibility to check if existing data can be overridden once new active data is pushed via CSV upload.

## Problem Description
Hi team,

For Tata Technologies (Company ID - 1102), employee data was previously being pushed via SFTP. However, due to issues on the client's end, they have now switched to CSV uploads for employee data updates.

Currently, the client uploads only the active associates through the CSV files. As a result, associates who have become inactive are not included in the latest uploads but continue to remain active in our system. This is causing discrepancies, as inactive associates are still appearing in the Active Employees Data Report.

We would like to check the feasibility of implementing the following solution:

Once a new CSV file is uploaded, the existing employee data should be automatically reconciled with the latest file.
Any associate record that is not present in the latest CSV upload should be automatically marked as inactive in the system.

Could you please review this requirement and let us know the best possible approach to address this scenario?

Thank you.
