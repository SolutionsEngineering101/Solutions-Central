How a rate is resolved

Every redemption converts points to money as amount = points ÷ pointsPerUnit. The rate is resolved in two steps:

  1. Explicit client rule — a per-client (sometimes per-country) override configured in the application, e.g. Wipro's 59-country map or Deloitte US at 1 pt = $1.
  2. Country default — if no rule exists, the country's default pointsPerUnit applies (see the Country defaults tab; values are 1, 10 or 100).

What the data shows

Zero contradictions. Across 2.19M redemptions (Nov 2024 – Jul 2026), every transaction where a known rule or default applies matches it exactly. The rate table in this tool is empirically validated, not just documented.
  • ~76% of clients (172 of 226) run 1 pt = 1 unit of local currency in every country — by far the dominant policy, and it covers ~95% of transaction volume.
  • 26 clients use 100 pts = 1 unit everywhere (LTIMindtree, Tata Communications, LPL Financial…). One (Bosch RBIN-style) uses 10 pts = 1.
  • 27 clients have genuinely custom per-country maps — Wipro is the largest (25 pts = $1, 38.9 pts = £1, 1.5 pts = ₱1…).

Cross-currency (FX-linked) redemptions

When an employee's points are pegged to their home currency but they redeem a card in a different currency (typically USD or EUR), the observed ratio floats with the FX rate of the day. Example: Amdocs Mexico is 1 pt = MX$1, so a USD card shows ≈17–19 pts per $1 — the MXN/USD rate. These rows carry the FX badge; the peg itself is still fixed.

Reading the badges

  • Rule — matches an explicit client override configured in the app.
  • Default — matches the country's default rate (no client override).
  • Observed — derived purely from redemption data; a stable flat rate but the client's ID couldn't be matched to a configured rule by name.
  • FX — cross-currency redemption; ratio ≈ peg × FX rate, varies daily.

Caveats

  • Rates are shown only for client-country pairs that exist in configuration or have at least one redemption — a country with zero activity for a client won't appear (the country default would apply).
  • Transaction counts indicate confidence: treat rates seen on <10 transactions with care.
  • Data window: redemption ledger through 30 Jun 2026; rules snapshot as deployed Jul 2026.
